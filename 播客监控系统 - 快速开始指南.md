# 播客监控系统 - 快速开始指南

## 第一步：获取您想监控的播客ID

在开始使用系统之前，您需要获取想要监控的播客在小宇宙平台上的唯一标识符（podcast_id）。

**操作步骤：**

1. 打开浏览器，访问 [小宇宙网站](https://www.xiaoyuzhoufm.com/)
2. 搜索并进入您想监控的播客主页（例如《随基漫步》）
3. 查看浏览器地址栏的URL，格式类似：
   ```
   https://www.xiaoyuzhoufm.com/podcast/6013f9f58e2f7ee375cf4216
   ```
4. 复制 `/podcast/` 后面的那串字符，这就是 `podcast_id`
   - 在上面的例子中，podcast_id 是：`6013f9f58e2f7ee375cf4216`

## 第二步：配置监控列表

打开项目目录下的 `config.json` 文件，找到 `podcasts` 数组部分：

```json
{
  "podcasts": [
    {
      "name": "随基漫步",
      "podcast_id": "在这里粘贴您获取的podcast_id",
      "category": "公募基金",
      "enabled": true
    },
    {
      "name": "知行小酒馆",
      "podcast_id": "6013f9f58e2f7ee375cf4216",
      "category": "泛财经",
      "enabled": true
    }
  ]
}
```

**配置说明：**

- `name`：播客的名称，用于在报告中显示
- `podcast_id`：从小宇宙URL中获取的ID
- `category`：播客分类（如"公募基金"、"泛财经"等）
- `enabled`：是否启用监控（`true` 启用，`false` 禁用）

**添加更多播客：**

只需在 `podcasts` 数组中添加新的配置块即可：

```json
{
  "name": "另一个播客",
  "podcast_id": "xxxxxxxxxxxxxxxxxxxxxxxx",
  "category": "投资理财",
  "enabled": true
}
```

## 第三步：运行系统

打开终端（命令行），执行以下命令：

```bash
# 进入项目目录
cd /home/ubuntu/podcast_monitor

# 运行监控程序
python3 main.py
```

系统将自动完成以下操作：
1. 爬取配置文件中所有启用的播客数据
2. 将数据存储到数据库中
3. 生成Word格式的日报

## 第四步：查看报告

运行完成后，在 `reports/` 目录下会生成一份以日期命名的Word文档：

```
reports/播客监控日报_2026-02-03.docx
```

报告包含以下内容：
- **数据概览**：监控播客总数、今日更新数、订阅数变化
- **详细数据表格**：每个播客的订阅数、变化情况、最新节目
- **分类统计**：按分类汇总的统计数据

## 常见问题

**Q: 如何设置每日自动运行？**

A: 您可以使用系统的定时任务功能（如Linux的cron）来实现每日自动执行。例如，每天早上8点运行：

```bash
# 编辑crontab
crontab -e

# 添加以下行（每天8:00执行）
0 8 * * * cd /home/ubuntu/podcast_monitor && python3 main.py
```

**Q: 为什么某个播客显示"缺少 podcast_id"？**

A: 这说明您在 `config.json` 中该播客的 `podcast_id` 字段为空或未填写。请按照第一步的说明获取正确的ID并填入。

**Q: 数据对比功能什么时候生效？**

A: 系统会自动对比今天和昨天的数据。第一次运行时由于没有历史数据，变化值会显示为0。从第二天开始，系统就能正确显示订阅数的增减情况。

**Q: 如何查看历史数据？**

A: 所有历史数据都存储在 `data/podcast_monitor.db` 数据库中。您可以使用SQLite工具（如DB Browser for SQLite）打开查看。
