# 播客监控与日报系统

## 1. 系统概述

本系统旨在自动监控小宇宙（xiaoyuzhoufm.com）平台上的指定播客节目，每日抓取其最新动态，包括订阅数、节目更新情况等，并自动生成一份精美的Word格式日报，方便您快速掌握所关注播客的表现。

## 2. 系统功能

- **自动数据抓取**：每日定时或手动执行，从网页端抓取播客的核心数据。
- **历史数据存储**：使用SQLite数据库持久化存储每日数据，为趋势分析和数据对比提供支持。
- **智能数据对比**：自动计算并展示订阅数相较于前一日的变化情况。
- **精美日报生成**：一键生成包含数据概览、详细表格和分类统计的Word（.docx）格式日报。
- **高度可配置**：通过简单的JSON配置文件，即可轻松添加、删除或禁用需要监控的播客。

## 4. 使用指南

### 4.1 配置监控列表

核心配置位于 `config.json` 文件中。您可以根据需求修改此文件来管理您的播客监控列表。

```json
{
  "podcasts": [
    {
      "name": "随基漫步",
      "podcast_id": "在此处填入播客ID",
      "category": "公募基金",
      "enabled": true
    },
    {
      "name": "知行小酒馆",
      "podcast_id": "6013f9f58e2f7ee375cf4216",
      "category": "泛财经",
      "enabled": true
    },
    {
      "name": "另一个播客",
      "podcast_id": "xxxxxxxxxxxxxxxxxxxxxxxx",
      "category": "其他分类",
      "enabled": false
    }
  ],
  // ... 其他设置
}
```

**如何添加新的播客？**

1.  在 `podcasts` 列表中，复制一个已有的播客配置块。
2.  修改 `name` 为您想监控的播客名称。
3.  **获取 `podcast_id`**：
    -   在小宇宙网站上打开该播客的主页。
    -   其URL类似于 `https://www.xiaoyuzhoufm.com/podcast/6013f9f58e2f7ee375cf4216`。
    -   URL中 `/podcast/` 后面的那串字符就是 `podcast_id`。
4.  修改 `category` 为该播客的分类。
5.  确保 `enabled` 设置为 `true`。

### 4.2 运行系统

系统配置了简单的一键执行方式。

```bash
# 进入项目目录
cd /home/ubuntu/podcast_monitor

# 运行主程序
python3 main.py
```

程序将自动完成**数据爬取** -> **数据存储** -> **日报生成**的全过程。

### 4.3 查看报告

每次成功运行后，最新的日报将保存在 `reports/` 目录下，并以日期命名，例如 `播客监控日报_2026-02-03.docx`。

## 5. 技术实现

- **爬虫核心**：使用 `requests` 库模拟浏览器请求，`BeautifulSoup` 解析HTML页面内容。
- **数据存储**：采用轻量级的 `SQLite` 数据库，无需额外配置，方便数据迁移和管理。
- **报告生成**：利用 `python-docx` 库动态创建和编辑Word文档，实现丰富的格式化输出。

## 6. 注意事项

- 请为您自己的播客《随基漫步》在 `config.json` 文件中填入正确的 `podcast_id`。
- 网页爬虫依赖于目标网站的页面结构，如果小宇宙网站进行大规模改版，爬虫脚本可能需要相应调整。
- 请遵守网站的 `robots.txt` 协议，并设置合理的请求延迟，避免对目标服务器造成过大压力。
